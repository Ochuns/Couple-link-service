# Feature Specification: Ochuna Link

**Feature Branch**: `001-ochuna-link`

**Created**: 2026-05-29

**Status**: Draft

**Input**: User description: "遠距離カップル向け再会カウントダウン・天気・距離共有Webアプリ「Ochuna Link」"

---

## Overview

Ochuna Link は、遠距離恋愛中のカップル2人をつなぐWebアプリです。次の再会までのカウントダウン、共有やることリスト、お互いの天気・距離の表示、そして過去の再会を記録する写真アルバム機能を提供します。「一緒にいる感覚」を日常の中で小さく積み重ねることを目的としています。

---

## Clarifications

### Session 2026-05-29

- Q: パートナーとの接続を解除した場合、共有データ（タスク・カウントダウン・写真）はどうなるか？ → A: 解除と同時にすべての共有データを即時削除する
- Q: パートナーが何かを更新したとき、相手への通知手段は？ → A: アプリ内通知のみ（アプリを開いているときのみリアルタイム同期）
- Q: タスクは作成後に編集・並べ替えできるか？ → A: テキスト編集可・並び替えなし（追加順固定）
- Q: パスワードを忘れたユーザーへのリカバリー手段は？ → A: メールにリセットリンクを送信
- Q: 1つの再会記録に添付できる写真の最大枚数は？ → A: 無制限（アカウント全体でストレージ上限を設定）

## User Scenarios & Testing *(mandatory)*

### User Story 1 - ペアリング設定 (Priority: P1)

パートナー同士がアカウントを作成し、招待コードを使って互いに接続（ペアリング）する。

**Why this priority**: ペアリングが完了しないと他のすべての機能が使えない。アプリの基盤となるフロー。

**Independent Test**: ペアリングだけを実装しても、「2人がつながった」という価値を独立して届けられる。

**Acceptance Scenarios**:

1. **Given** ユーザーが新規登録済みでパートナーが未接続の状態で、**When** ユーザーが招待コードを発行し、パートナーがそのコードを入力すると、**Then** 2人のアカウントがペアリングされ、共有ダッシュボードが表示される。
2. **Given** 無効な招待コードを入力した場合、**When** 接続を試みると、**Then** エラーメッセージが表示され、接続は完了しない。
3. **Given** すでにペアリング済みのユーザーが、**When** アプリを開くと、**Then** 共有ダッシュボードに直接遷移する。

---

### User Story 2 - 再会カウントダウン (Priority: P2)

次に会う日付を登録し、カウントダウンを2人で共有する。

**Why this priority**: アプリの感情的コアであり、遠距離特有の「あと何日」という体験を直接提供する。

**Independent Test**: カウントダウン機能だけを実装しても、次に会う日への期待感を共有するという価値が届く。

**Acceptance Scenarios**:

1. **Given** ペアリング済みのカップルが、**When** どちらかが次の再会日を設定すると、**Then** 2人のダッシュボードに「あと○日○時間」のカウントダウンがリアルタイムで表示される。
2. **Given** 再会日が設定済みの状態で、**When** 再会日を更新すると、**Then** カウントダウンが新しい日付に基づいて即時更新される。
3. **Given** 再会日が未設定の状態で、**When** ダッシュボードを表示すると、**Then** 「次の再会日を設定しよう」という案内が表示される。
4. **Given** 再会日が過去の日付になった場合、**When** ダッシュボードを表示すると、**Then** 「再会おめでとう！次の日程を設定しよう」というメッセージが表示される。

---

### User Story 3 - 天気・距離の確認 (Priority: P3)

お互いの現在地の天気と、2人の間の距離をダッシュボードで確認する。

**Why this priority**: 遠距離特有の「今あなたのところはどんな天気？」という共感ポイントを視覚化する。

**Independent Test**: 天気・距離表示だけを実装しても、相手の今の環境を知るという価値が独立して届く。

**Acceptance Scenarios**:

1. **Given** 両パートナーが都市を設定済みの状態で、**When** ダッシュボードを表示すると、**Then** それぞれの都市名・現在の天気・気温が並べて表示される。
2. **Given** 都市が設定済みの状態で、**When** ダッシュボードを表示すると、**Then** 2人の間のおおよその距離（km）が表示される。
3. **Given** どちらかの都市が未設定の場合、**When** ダッシュボードを表示すると、**Then** 未設定側に「都市を設定してください」という案内が表示される。
4. **Given** 天気APIが取得できない場合、**When** ダッシュボードを表示すると、**Then** 「天気情報を取得できません」というフォールバックメッセージが表示される。

---

### User Story 4 - やることリストの共有 (Priority: P4)

再会までにやりたいこと・準備することをリストにして2人で共有・管理する。

**Why this priority**: 「次に会うまでに〇〇しよう」という共同体験を作り出し、日常的な接点を増やす。

**Independent Test**: リスト機能だけを実装しても、共同タスク管理という価値が独立して届く。

**Acceptance Scenarios**:

1. **Given** ペアリング済みのカップルが、**When** どちらかがタスクを追加すると、**Then** 両方のダッシュボードに即時反映される。
2. **Given** タスクが表示されている状態で、**When** チェックボックスをタップすると、**Then** タスクが完了状態になり、見た目が変わる。
3. **Given** タスクが完了状態の場合、**When** 削除ボタンを押すと、**Then** タスクが一覧から削除される。
4. **Given** 既存のタスクがある状態で、**When** タスクのテキストを編集して保存すると、**Then** 更新された内容が両パートナーのリストに即時反映される。

---

### User Story 5 - 再会写真の記録 (Priority: P5)

実際に会った後、その再会の写真をアップロードして「再会アルバム」として記録・閲覧する。

**Why this priority**: 過去の再会を振り返ることで、距離を超えたつながりの証を残せる。

**Independent Test**: アルバム機能だけを実装しても、思い出を記録・閲覧するという価値が届く。

**Acceptance Scenarios**:

1. **Given** ペアリング済みのカップルが、**When** どちらかが再会の記録（日付・写真・一言コメント）を投稿すると、**Then** アルバムページに表示される。
2. **Given** 複数の再会記録がある場合、**When** アルバムページを開くと、**Then** 新しい順に一覧が表示される。
3. **Given** アップロードする写真が10MBを超える場合、**When** 投稿しようとすると、**Then** サイズ超過のエラーメッセージが表示される。

---

### Edge Cases

- パートナーが未接続の状態でカウントダウンや天気を設定しようとした場合の動作
- 再会日が当日に設定された場合のカウントダウン表示（「0日」表示）
- ネットワーク接続が切れている状態でのアプリ動作（オフライン時のフォールバック）
- 同一ユーザーが複数デバイスで同時接続した場合のデータ整合性
- 招待コードの有効期限切れ（長期間未使用のコードへの対応）
- 一方のユーザーが接続解除操作中に、パートナーが共有データを閲覧・更新しようとした場合の整合性
- 接続解除後に同じ2人が再ペアリングしようとした場合の扱い（新規カップルとして扱う）

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムはユーザーがメールアドレスとパスワードで新規登録・ログインできること
- **FR-019**: システムはパスワードを忘れたユーザーに対し、登録メールアドレス宛にリセットリンクを送信できること
- **FR-002**: システムはログイン済みユーザーに対して固有の招待コードを発行できること
- **FR-003**: システムは有効な招待コードを入力した2人のユーザーをペアリングし、共有ダッシュボードを作成すること
- **FR-004**: ペアリング済みユーザーは次の再会日を設定・更新できること
- **FR-005**: システムは再会日までのカウントダウン（日・時間・分）をリアルタイムで表示すること
- **FR-006**: ユーザーは自身の都市名を設定・変更できること
- **FR-007**: システムは両パートナーの設定都市の現在の天気・気温を定期的に取得して表示すること
- **FR-008**: システムは両パートナーの設定都市間のおおよその直線距離を計算・表示すること
- **FR-009**: ペアリング済みユーザーは共有タスクリストにアイテムを追加・編集・完了・削除できること（表示順は追加順固定、手動並び替えは対象外）
- **FR-010**: タスクリストの変更はリアルタイムで両パートナーに反映されること
- **FR-011**: ユーザーは再会の記録（日付・枚数制限なしの写真・コメント）を投稿できること
- **FR-012**: 再会記録はアルバムとして新着順に一覧表示されること
- **FR-013**: システムはユーザーが表示名とプロフィール画像を設定できること
- **FR-014**: ペアリング済みユーザーはいつでもパートナーとの接続を解除できること
- **FR-015**: 接続解除と同時に、すべての共有データ（タスク・カウントダウン・写真・再会記録）は即時かつ完全に削除されること
- **FR-016**: 接続解除後、両ユーザーのアカウントは維持されるが、ペアリング前の状態（新規招待コード発行可能）に戻ること
- **FR-017**: 共有データの更新（タスク追加・再会日設定など）はアプリ内のみで反映され、プッシュ通知・メール通知は送信しないこと
- **FR-018**: アプリを開いている両パートナーに対し、共有データの変更はリアルタイムで反映されること（ページリロード不要）

### Key Entities *(include if feature involves data)*

- **User（ユーザー）**: 表示名、メールアドレス、プロフィール画像、設定都市
- **Couple（カップル）**: 2人のユーザーの接続関係、招待コード、次の再会日
- **Task（タスク）**: タイトル、完了状態、作成者、作成日時
- **Reunion（再会記録）**: 再会日付、コメント、関連写真、投稿者
- **WeatherSnapshot（天気情報）**: 都市名、気温、天気状態、最終取得日時

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 新規ユーザーがアカウント作成からペアリング完了まで3分以内に完了できる
- **SC-002**: カウントダウンはページリロードなしに1分ごとに自動更新される
- **SC-003**: 天気情報は最後の取得から10分以内のデータが表示される
- **SC-004**: タスクの追加・完了・削除がパートナー側に5秒以内に反映される
- **SC-005**: 5MB以下の写真のアップロードが10秒以内に完了する
- **SC-006**: ダッシュボードが3秒以内に完全にロードされる（標準的なモバイル回線環境）
- **SC-007**: スマートフォン・タブレット・PCのいずれでも主要機能が問題なく操作できる

---

## Assumptions

- カップルは常に2名で構成される（3名以上のグループには対応しない）
- 位置情報はGPS取得ではなく、ユーザーが都市名を手動で入力する（プライバシー配慮）
- 距離は都市間の直線距離で表示し、移動経路の距離は考慮しない
- 天気情報は都市単位で取得する（詳細な地区レベルは対象外）
- 写真は再会記録に紐づけて保存する（スタンドアロンの写真共有機能は対象外）
- v1は日本語のみをサポートする
- モバイルファーストのレスポンシブWebアプリとして実装する（ネイティブアプリは対象外）
- ユーザーは安定したインターネット接続環境を持つことを前提とする
- 招待コードの有効期限は24時間とする
- 写真は枚数無制限だが、1アカウントあたりのストレージ上限を設ける（上限値はv1リリース前に決定）
