# Feature Specification: 共有カレンダー & ごはんシェア

**Feature Branch**: `002-calendar-meal-share`

**Created**: 2026-05-29

**Status**: Draft

**Input**: User description: "カレンダーのようなものも追加して、一緒に食べたご飯を写真で掲載できるようにしたい（Ochuna Link への機能追加）"

---

## Overview

Ochuna Link に2つの新機能を追加します。

1. **共有カレンダー**: カップルがビデオデートや記念日などの予定を登録・確認できるシンプルなスケジュール管理機能
2. **ごはんシェア**: 「一緒に食べた（バーチャルで同時に食べた）ご飯」の写真と一言メモを投稿・閲覧できる日常共有機能

どちらも「遠距離でも毎日つながっている感覚」をさらに深めることを目的としています。

---

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 共有カレンダーへの予定登録 (Priority: P1)

どちらかのパートナーが2人の予定（ビデオデート、記念日、再会など）をカレンダーに追加し、両方が確認できる。

**Why this priority**: カレンダー機能のコアであり、これだけで「次の予定を共有する」価値を届けられる。

**Independent Test**: 予定の追加と一覧表示だけ実装しても、スケジュール共有という価値が独立して届く。

**Acceptance Scenarios**:

1. **Given** ペアリング済みのカップルが、**When** どちらかが日付・タイトル・任意の説明を入力して予定を登録すると、**Then** 2人のカレンダーに即時反映される。
2. **Given** カレンダーに複数の予定がある状態で、**When** カレンダーページを開くと、**Then** 今月の予定が一覧・カレンダー形式で表示される。
3. **Given** 予定を作成したユーザーが、**When** その予定を編集・削除すると、**Then** 変更が両方のカレンダーに即時反映される。
4. **Given** 当日または翌日に予定がある場合、**When** ダッシュボードを表示すると、**Then** 直近の予定が「もうすぐの予定」としてハイライト表示される。

---

### User Story 2 - ごはんシェアの投稿 (Priority: P2)

食事の写真と一言メモを投稿し、「バーチャルで一緒に食べている感覚」をパートナーと共有する。

**Why this priority**: 日常の小さなつながりを作る機能で、毎日使われる可能性が高い。

**Independent Test**: ごはん投稿と閲覧だけ実装しても、日常の食事を共有するという価値が届く。

**Acceptance Scenarios**:

1. **Given** ペアリング済みのカップルが、**When** どちらかが食事の写真・一言メモ・任意の「一緒に食べたよ」フラグを投稿すると、**Then** パートナーのごはんフィードに表示される。
2. **Given** ごはんフィードに複数の投稿がある状態で、**When** フィードを開くと、**Then** 新しい順に2人の投稿が時系列で表示される。
3. **Given** パートナーの投稿が表示されている状態で、**When** 絵文字リアクションを送ると、**Then** 投稿者のリアクション表示が更新される。
4. **Given** 自分の投稿がある場合、**When** 削除ボタンを押すと、**Then** 投稿がフィードから削除される。

---

### User Story 3 - カレンダーと予定タイプの管理 (Priority: P3)

記念日・ビデオデート・ごはんデートなど、予定にタイプを付けて管理する。

**Why this priority**: 予定の種類を視覚的に区別することで、カレンダーの可読性と感情的価値が高まる。

**Independent Test**: 予定タイプ機能だけ実装しても、視覚的な整理という価値が届く。

**Acceptance Scenarios**:

1. **Given** 予定を追加する際に、**When** 予定タイプ（記念日・ビデオデート・ごはんデート・その他）を選択すると、**Then** カレンダー上でタイプごとに色分け表示される。
2. **Given** カレンダーに複数タイプの予定がある状態で、**When** 特定のタイプでフィルタリングすると、**Then** 該当タイプの予定のみ表示される。

---

### Edge Cases

- 同じ日付に複数の予定が重なった場合のカレンダー表示
- 過去の日付に予定を登録しようとした場合の扱い（許容 or 警告）
- ごはん写真の投稿時に写真が選択されなかった場合（テキストのみ投稿の可否）
- ネットワーク接続が不安定な状態での写真アップロード中断時の挙動
- 接続解除時のカレンダー・ごはんシェアデータの削除（既存のFR-015と同様）

## Requirements *(mandatory)*

### Functional Requirements

**共有カレンダー**

- **FR-CAL-001**: ペアリング済みユーザーは共有カレンダーに予定（日付・タイトル・任意の説明・予定タイプ）を追加できること
- **FR-CAL-002**: 予定タイプは「記念日」「ビデオデート」「ごはんデート」「その他」の4種類から選択できること
- **FR-CAL-003**: カレンダーは月単位で表示でき、前後の月に移動できること
- **FR-CAL-004**: 予定の作成者はいつでも予定を編集・削除できること
- **FR-CAL-005**: カレンダーの変更（追加・編集・削除）はアプリ内でリアルタイムに両パートナーへ反映されること
- **FR-CAL-006**: 当日・翌日の予定はダッシュボードの「直近の予定」セクションにハイライト表示されること
- **FR-CAL-007**: 接続解除時、すべての共有カレンダーデータも即時削除されること

**ごはんシェア**

- **FR-MEAL-001**: ペアリング済みユーザーは食事の写真・一言メモ（任意）・「一緒に食べたよ」フラグ（任意）を投稿できること
- **FR-MEAL-002**: 投稿した写真は10MB以内であること（超過時はエラーメッセージを表示）
- **FR-MEAL-003**: ごはんフィードは2人の投稿を新着順に時系列で表示すること
- **FR-MEAL-004**: 投稿に対して絵文字リアクション（1投稿につき1リアクション）を送れること
- **FR-MEAL-005**: 投稿者は自分の投稿をいつでも削除できること
- **FR-MEAL-006**: ごはんフィードの新着投稿はアプリ内でリアルタイムに反映されること
- **FR-MEAL-007**: 接続解除時、すべてのごはんシェアデータも即時削除されること

### Key Entities *(include if feature involves data)*

- **CalendarEvent（カレンダー予定）**: 日付、タイトル、説明（任意）、予定タイプ、作成者、作成日時
- **EventType（予定タイプ）**: 記念日、ビデオデート、ごはんデート、その他
- **MealPost（ごはん投稿）**: 写真、メモ（任意）、「一緒に食べたよ」フラグ、投稿者、投稿日時
- **Reaction（リアクション）**: 絵文字、投稿ID、リアクション者

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: カレンダーへの予定追加がパートナー側に5秒以内に反映される
- **SC-002**: カレンダーページが3秒以内に完全表示される（1ヶ月分の予定）
- **SC-003**: ごはん投稿がパートナーのフィードに5秒以内に表示される
- **SC-004**: 5MB以下の写真のアップロードが10秒以内に完了する
- **SC-005**: 予定の追加から確認完了まで1分以内に操作できる
- **SC-006**: スマートフォン・タブレット・PCのいずれでもカレンダーとフィードが問題なく操作できる

---

## Assumptions

- カレンダーは過去の日付への予定登録も許容する（記念日の記録用途を考慮）
- ごはん投稿には写真が必須（テキストのみの投稿は対象外）
- 絵文字リアクションは1投稿につき各ユーザー1つのみ（変更・取り消し可能）
- カレンダーのリアルタイム同期は既存のアプリ内通知のみポリシー（FR-017）に従う
- ごはんフィードは既存の再会アルバムとは独立した別のセクションとして表示する
- この機能はOchuna Link（`001-ochuna-link`）のペアリング基盤を前提として動作する
